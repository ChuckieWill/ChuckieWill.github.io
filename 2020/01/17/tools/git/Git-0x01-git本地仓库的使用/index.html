2 Git仓库文件状态管理

2.1 Git文件状态

2.1.1文件状态概述

每一个文件有两种状态：已跟踪 或 未跟踪。 已跟踪的文件是指那些被纳入了版本控制的文件，在上一次快照（git commit提交）中有它们的记录，在工作一段时间后， 它们的状态可能是未修改，已修改或已放入暂存区。

  • 未跟踪（Untracked）———git add
  • 已跟踪 —————————-git commit
    • 未修改（Unmodified）
    • 已修改(Modified)
    • 已放入暂存区(Staged) —git add

2.2.4 撤销或删除暂存区文件

撤销暂存区的某文件

  • 将已保存到暂存区的文件撤销保存，即将文件移除暂存区，使文件状态处于未加入暂存区的状态
  • 即取消最近一次加入暂存区之前的修改

删除暂存区的某文件（不再跟踪某文件）

  • 只把文件从 Git 仓库中删除（即从暂存区域移除），但仍然保留在当前工作目录中。 即让文件保留在磁盘，但Git不再继续跟踪。

git reset HEAD和git rm --cached的区别

  • git rm – cached 是从stage（暂存区） 里面删除文件，当你提交（commit）之后文件就不再被跟踪了。
    • 该文件不再被跟踪
  • git reset HEAD : 回退暂存区里的文件（还原为HEAD commit里面该文件的状态），会撤销从上一次提交（commit）之后的一些操作。
    • 该文件仍然被跟踪，只是这次的修改没有加入暂存区
  • 如果是对于新增文件，这两个操作是等效的。
